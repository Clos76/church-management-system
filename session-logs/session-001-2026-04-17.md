# Session 001 — 2026-04-17

## Summary
Full 8-phase architecture refactor based on `systemAudits/audit1.md` findings.

## Phases Completed

### Phase 1 — Architecture Enforcement
- Removed all direct `supabase.from()` calls from UI pages
- Created `UserService` (`getUsers`, `updateUserRole`, `getAssignableUsers`)
- Created `ReportService` (`getDashboardStats` — 6 parallel queries)
- Fixed `window.location.origin` ReferenceError in `event-service.ts`

### Phase 2 — Security Hardening
- Created `src/middleware.ts` — session refresh + redirect unauthenticated users
- Created `src/lib/auth/require-role.ts` — server-side role guard for layouts
- Created `AdminShell.tsx` — client component receiving `AuthUser` prop
- Converted `admin/layout.tsx` from 336-line client component to 12-line Server Component
- Created `src/app/api/users/[id]/route.ts` — PATCH with server-side admin verification
- Added `leader/layout.tsx` with `requireRole(["admin", "event_leader"])`

### Phase 3 — Data Safety & Consistency
- Added pagination to `MemberService`, `EventService`, `PaymentService`, `RegistrationService`
- Fixed registration race condition — DB unique constraint `uq_member_event(member_id, event_id)` + `23505` error handling
- Fixed calendar N+1 — single embedded query instead of per-row queries
- Migration: `supabase/migrations/20260417_phase3_constraints_indexes.sql`

### Phase 4 — Service Layer Completion
- Created `TimelineService` — `addEntry()` + `getTimeline()`
- Created `TaskService` — full CRUD with status tracking
- Created `CommunicationService` — `sendEmail()` + `getEmailLogs()`
- Created `src/app/api/communications/send/route.ts` — server-side Resend + `email_logs`
- Migration: `supabase/migrations/20260417_phase4_new_tables.sql` (activity_logs, tasks, email_logs)

### Phase 5 — Domain Events Foundation
- Created `src/lib/services/types.ts` — shared `ServiceResult<T>`, `PaginatedResult<T>`
- Created `src/lib/events/domain-events.ts` — typed `DomainEvent` union (12 event types)
- Created `src/lib/events/event-bus.ts` — in-memory `DomainEventBus` (fire-and-forget)
- Created `src/lib/events/setup.ts` — registers TimelineService handlers for all events
- Emitters added: `member.created`, `member.statusChanged`, `event.registered`, `task.created`, `task.completed`, `donation.completed`
- `AdminShell.tsx` and `LeaderShell.tsx` import `setup.ts` to register handlers on mount

### Phase 6 — Timeline System
- Verified: `activity_logs` only written by `TimelineService.addEntry()`, only called from `setup.ts`
- Added `email.sent` event emission to `CommunicationService.sendEmail()`
- Added `@internal` JSDoc to `TimelineService.addEntry()` as architectural guard

### Phase 7 — Performance & Scalability
- Converted `admin/page.tsx` from Client Component to Server Component
- Created `report-service-server.ts` with `unstable_cache` (1-hour TTL, key: `dashboard-stats`)
- Capped unbounded `pendingPayments` query with `.limit(500)`
- Fixed remaining `ServiceResult` import from `./member-service` → `./types`

### Phase 8 — Code Quality Cleanup
- Created `src/components/ui/Toast.tsx` — `ToastProvider` + `useToast()` hook
- Wired `ToastProvider` into `AdminShell` and `LeaderShell`
- Created `admin/loading.tsx`, `admin/error.tsx`, `leader/loading.tsx`, `leader/error.tsx`
- Replaced all 34 `alert()` calls across 12 files with `toast.success/error/info()`
- Replaced both `confirm()` calls with inline state dialogs (no browser confirm)
- Public register page uses inline `submitError` banner (no provider needed)
